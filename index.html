    - 언어: Java(JDK 11.0.19), HTML/CSS, JavaScript

    - 서버: AWS EC2

    - 프레임워크: Spring boot(2.7.17), JPA

    - DB: MySQL(8.0)

    - IDE: IntelliJ

    - API: Google/Naver/Kakao Login API, Daum 주소 검색

    - 라이브러리: lombok, thymeleaf(3.1.0), Spring Security, OAuth2

    💡 기록 및 회고

    1. 어려웠던 부분

    - 스프링 시큐리티에서 회원 구분을 아이디로 하는데 사용자가 설정한 아이디로 바꾸고 싶어서 그렇게 구현하도록 프로그래밍했다. 그런데 멤버 아이디를 외래키로 받는 다른 기능 구현들에서 막혀서 결국 다시 이메일로 설정했다.

    - OAuth에 대해 처음 알고 구현하다 보니 각 사이트에서 받아오는 회원 고유의 값과 속성이 달라서 그것을 아는 데에 시간이 좀 걸렸다.

    2. 개선할 부분

    - 회원 탈퇴의 경우 스프링 시큐리티를 통하여 가입한 회원은 탈퇴가 가능하나, OAuth2를 통하여 SNS 로그인으로 가입한 회원은 탈퇴 시 API를 통하여 연결을 끊어야 했다.

    - 프로젝트 마감 시간이 임박하여 그것을 구현하지 못했다.

    3. 추가하고 싶은 기능

    팀 프로젝트 기간이 짧아서 프로젝트에서 결제 기능을 구현하지 못했는데 맡은 부분과 상관없이 결제 부분을 구현해보고 싶다.

    💡 기록 및 회고

    1. 어려웠던 부분

    - 풀 페이지 CSS 기능을 활용하여 스크롤 시 페이지를 이동하고 싶었는데 그렇게 구현하니 헤더와 푸터 안에 있는 페이지에 적용되어 스크롤이 여러 개 생겼다. 보기에 좋지 않고 복잡하여 구현하지 못했다.

    - 모달 팝업 구현하면서 js의 선택자와 순서 적용법을 잘 몰라 헤매다 구현했다.

    - 모달 팝업 실행 시 모달 팝업이 아닌 다른 곳을 클릭하면 모달이 닫치는 기능을 구현하고 싶었는데 하지 못했다.

    - 미디어쿼리를 적용하여 반응형 페이지를 만들었다. 모바일일 때 내비게이션바가 간소화되도록 구현하는 데에 익숙지 않아 시간이 좀 걸렸다.

    2. 개선할 부분

    - 모달 팝업 실행 시 스크롤이 되지 않고 페이지를 고정하고 싶다.

    - 모바일일 때 메뉴 클릭시, 모달 팝업 실행 시 뒷 배경을 누르면 닫히도록 구현하고 싶다.

    3. 추가하고 싶은 기능

    - 메시지 폼을 만들어서 제목과 내용을 적어 전송 버튼을 누르면 DB에 저장되어 내가 볼 수 있도록 하는 기능을 추가하고 싶다.
